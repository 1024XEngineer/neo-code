# Utils 模块

## 角色定位

Utils 提供跨模块复用的通用辅助能力，目标是减少重复实现并统一低层约束表达。

## [CURRENT] 实现基线

- 当前仓库没有独立 `internal/utils` 包。
- 工具函数分散在 `runtime/context/tools/security/config` 等模块内部。
- 当前可见能力类型包括：
  - 路径规范化与安全检查辅助。
  - 文本裁剪与格式化辅助。
  - 会话 ID、时间戳、文件命名等基础辅助。
  - compact 摘要协议相关辅助（模板、校验、截断）。

## [PROPOSED] 目标态

- 统一建立独立 utils 契约与实现边界，优先覆盖运行时辅助能力：
  - token 估算。
  - 摘要/截断辅助。
  - ID 生成与时间抽象。
  - 路径规范化。
  - 轻量序列化与负载收敛。
- 按“纯函数优先、无副作用优先”设计，避免把业务流程迁入 utils。

## 上下游边界

- 上游：runtime/context/tools/provider/config。
- 下游：标准库与无业务语义的基础工具实现。
- 约束：utils 不负责编排，不直接访问 provider/tools 执行链，不持久化会话。

## 联调注意事项

- 当前没有稳定 `utils` 公共接口，不应假设存在统一调用入口。
- 新增 utils 后需保持行为与既有模块内 helper 一致，避免语义分叉。
