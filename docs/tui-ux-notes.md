# TUI 交互与提示策略

## 默认提示策略

- 底部状态栏只保留高价值快捷键，默认不再展示 `Ctrl+V paste` 这类显而易见的提示。
- 完整快捷键说明收敛到 `?` 帮助面板，减少常驻信息噪音。
- composer 和空态说明优先提醒真正有价值的行为，例如会话切换、代码复制和帮助入口。

## 代码块复制交互

- 代码块标题栏保留 `Copy` 入口，支持鼠标直接点击复制。
- 键盘复制行为保持不变：`y` 复制当前代码块，`Y` 复制当前消息。
- 复制仍写入系统剪贴板，不依赖终端原生文本选中。

## 渲染约束

- badge / action 组件必须保持单行渲染，不能把多行边框组件直接拼进单行标题区域。
- 所有依赖 badge / action 宽度的点击热区都必须基于实际单行渲染宽度计算。
- 代码块头部右侧操作区需要优先保证可见，避免标题过长时把复制入口挤掉。
